# Iteration 18 — Armor + Shell + Boss

## 1. Що додано
Три нові типи блоків з різними механіками "товстошкірості":
- **Armored** (бронза) — має armor 3. Кожна атака зменшується на 3 (мінімум 0.5)
- **Shell** (пісочний) — двошаровий: спочатку зовнішня шкаралупа, потім внутрішня. Після пробиття shell — inner регенерує 40%/sec
- **BOSS** (фіолетовий) — гігантський (×1.6 розмір), HP × 5, armor 2, **контратакує** кожні 6 сек: знищує найслабшу кирку на сітці. З'являється з ур.5

## 2. Що змінилось у файлах
- `BlockTypeData` — нові поля: `armor`, `innerHPMultiplier`, `shellRegenPerSec`, `isBoss`, `bossCounterAttackInterval`, `bossSizeMultiplier`
- `Block` — рефакторинг TakeDamage:
  - Spread armor віднімається з damage (мін 0.5)
  - Якщо shellActive — damage йде у shellHP, після пробиття активна швидка регенерація inner HP
  - Boss блок у Update перевіряє bossCounterAttackInterval і викликає BossCounterAttack що видаляє найслабшу кирку
- `Block.Setup` — ініціалізує shell/inner HP, boss розмір
- `Block.BossCounterAttack` — знаходить найслабшу кирку (з найнижчим Level), руйнує її
- `currentScale` поле — зберігає масштаб боса для коректних PunchScale анімацій
- Editor оновлює BlockConfig (8 типів) і LevelConfig (Armored з ур.1, Shell з ур.4, Boss з ур.5)

## 3. Editor скрипт

`Tools / Merge Mining / (Iteration 18) Update ALL`

## 4. Розкидка по рівнях
- **Ур.1-2**: Stone, Diamond, Explosive, Armored. Без Iron/Boss (Iron — мало хто пробиває lvl1 кирку, Boss — занадто страшно для початку)
- **Ур.3**: + Iron + Healer
- **Ур.4**: + Shell блоки (двошарові)
- **Ур.5+**: Boss блоки

## 5. Як тестувати

### Armored:
1. Запусти Level 1 → побачиш бронзовий Armored блок
2. Купи кирку lvl1 (damage 1) → атакує Armored → блок отримує 0.5 damage (мін), а не 1 - 3 = -2
3. Доводиться мержити вище, щоб кожна атака реально проходила

### Shell:
4. Запусти Level 4 → у послідовності є Shell блок (пісочний)
5. Б'єш — HP-бар йде від 100% вниз ШВИДКО — це shellHP
6. Коли shell зломаний — блок темніє на колір darkColor, бар повертається до повного 100% (це inner HP, інший)
7. Дальше треба пробивати inner. **Якщо припиниш атаки на 0.6s+ — inner регенерує 40%/sec**. Тобто треба добивати швидко

### Boss:
8. Запусти Level 5 → у послідовності є BOSS блок (фіолетовий, в 1.6 рази більший за звичайний)
9. HP × 5 від рівневого, armor 2, не сильно тримається
10. Кожні 6 секунд боса — спалах червоним, камера трясеться, **зникає одна з твоїх кирок (найслабша)**
11. Якщо боса довго не вб'єш — всі кирки помруть → DEFEAT

### Стратегія:
12. На пізніх рівнях боса треба добивати ПЕРШИМ
13. Shell — теж пріоритет, інакше регенерує
14. Healer — також серед перших

## 6. Очікуваний результат
- Armored — пробуджує необхідність мерджа (lvl1 кирка проти armor 3 = 0.5 damage = безсилля)
- Shell — змушує сконцентрованих атак, не розкидатися
- Boss — створює "контр-таймер" і змушує виставляти багато кирок

## 7. Відомі обмеження
- Boss block не має окремої гігантської HP-бар над собою — звичайний
- Якщо `blockSequence` коротший за `blocksToDestroy`, ті ж типи повторюються (включаючи Boss — може зустрітися кілька разів)
- ShellHP візуально рендериться у тому ж hpBar що й MaxHP — гравець не бачить два бари
