# Iteration 21 — Block Rewards + Battle-Phase Shop

## 1. Що додано
- **Блоки знов дають монети** при знищенні (через анімацію burst)
- **Нагорода залежить від типу блоку** — кожен має свій rewardMultiplier
- **Магазин SHOP працює у фазі Battle** — можна купувати кирки під час бою на свіжозароблені coins

## 2. Нагороди за типи блоків
Базова нагорода = 3 coins, помножена на rewardMultiplier типу:
- **Stone** (rewardMult 1.0) → 3 coins
- **Iron** (1.3) → 4 coins
- **Diamond** (2.0) → 6 coins
- **Explosive** (1.5) → 5 coins
- **Healer** (1.2) → 4 coins
- **Armored** (1.4) → 4 coins
- **Shell** (1.8) → 5 coins
- **BOSS** (4.0) → 12 coins

## 3. Що змінилось у файлах
- `BlocksRowManager.SpawnBlockAt` — `reward = Mathf.Max(1, RoundToInt(3 * type.rewardMultiplier))` замість `0`
- `BlocksRowManager.OnBlockDestroyed` — викликає `FlyEffectSpawner.Instance.FlyCoins(pos, reward, 5)` (5 летящих монеток до HUD-icon з прибуттям). Fallback: якщо немає FlyEffectSpawner, додає coins одразу
- `ShopController.RefreshInteractable` — тепер shop активний у Setup ТА Battle фазах (раніше тільки Setup)

## 4. Editor скрипт

Не потрібен — це чисто code changes. Можеш натиснути `Tools / Merge Mining / (Iteration 21) Info` щоб побачити підтвердження.

## 5. Як тестувати

1. Запусти Game scene
2. У фазі Setup — як завжди, 45 стартових coins на ур.1
3. Купи 2-3 кирки, тапни START
4. Битва починається. **Коли вб'єш блок** — з нього вилітають 5 жовтих монеток до coin-iconки в HUD
5. Coin counter у HUD росте. Дивись цифру справа зверху
6. **Кнопка SHOP залишається активною** — можна тапати її в Battle. За зароблені coins можна купувати нові кирки прямо під час бою
7. Стратегія: на ур.5+ де HP блоків великий і Boss б'є кирки — гравець може утримувати "запас живої сили" купуючи нові кирки під час бою на зароблені coins

## 6. Очікуваний результат
- Гра динамічніша: гравець активно тапає SHOP у Battle коли збив якихось блоків
- Більша глибина: чи відразу мерджити чи купувати щоб тримати лінію
- Boss блок (12 coins) дає трохи економіки коли його ламаєш — приємний кікбек
- Анімація летящих coins робить гру "соковитою"

## 7. Відомі обмеження
- Покупки в Battle — це lvl 1 кирки. Гравець може мерджити їх з існуючими, тому це досі стратегічно цінне
- Якщо гравець обходить блоки повільно — coins вистачить на пару нових кирок, не на хвилю нових
- При Victory попап з'являється одночасно з прильотом останніх coins у HUD (FlyCoins на 0.6s)
